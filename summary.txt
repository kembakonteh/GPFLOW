## GPflow — Project Summary

**What it is:** A shipping/freight logistics web app for managing trips between the US and Gambia (and potentially other routes). Operators manage trips; customers book packages.

**Stack:**
- Frontend: React + TypeScript (Vite), in /frontend/src
- Backend: Python/FastAPI, in /backend/app
- Docker Compose for local dev

---

### Core Domain Concepts

- **Trips** — Either inbound (Gambia → US) or outbound (US → Gambia). Have a carrier, rate, departure/arrival dates, and a destination location.
- **Bookings** — Packages attached to a trip. Can be walk-in or mail-in. Go through states: created → weighed → departed → arrived → confirmed/delivered.
- **Rates** — Stored in kg in the DB, displayed in lb on the frontend. Conversion: rate_per_lb = rate_per_kg / 2.20462.
- **Delivery types** — operator_delivers (operator mails the package via USPS/UPS) vs drop-off.

---

### Key Pages & Components

- DashboardPage.tsx — Operator view, lists trips and bookings
- BookingFormPage.tsx — Customer-facing booking form
- PublicTripPage.tsx — Public trip info + booking entry point
- BookingConfirmedPage.tsx — Post-booking confirmation
- TrackingPage.tsx — Package tracking
- modals/ — ScanModal, ArrivedModal, DepartedModal, WeighModal, etc.
